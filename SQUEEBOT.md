# Squeebot - Discord Bot Project Plan

A Discord bot that roleplays as **Squee, the Immortal** from Magic: The Gathering using a lightweight LLM with RAG-grounded character voice.

---

## Overview

Squee is a cowardly, accident-prone, immortal goblin from the Weatherlight crew. The bot responds when mentioned in Discord with in-character replies generated by Gemini Flash, grounded by a dataset of 10,000 Squee voice lines and per-user "goblin notes" memory.

**Server scope**: 1 Discord server, 3 channels, ~20 active users.
**Trigger**: @mention only (not keyword-based).

## Hosting Strategy

| Component | Service | Cost |
|-----------|---------|------|
| **Bot process** (always-on WebSocket) | Fly.io free tier (3 free VMs, 256MB RAM each) | $0/mo |
| **LLM inference** | Google Gemini Flash free tier (1,500 req/day) | $0/mo |
| **Fine-tuning** (future, optional) | Modal ($30 free credits/month) | $0/mo |
| **Fallback bot hosting** | Railway (if Fly.io free tier changes) | ~$5/mo |

**Expected usage**: ~50-100 requests/day, well within all free tiers.

### Why These Choices

- **Fly.io** over Oracle Cloud: simpler DX, good free tier, easy deploys.
- **Gemini Flash** over other APIs: most generous free tier (1,500 req/day), high quality for character roleplay, free.
- **Modal** reserved for future fine-tuning experiments only (serverless GPU, not suited for persistent bot hosting).
- Discord bots require a persistent WebSocket connection, ruling out pure serverless (Vercel Functions, Lambda, etc.).

## Interaction Flow

On each @mention:

1. **Receive** - Bot detects mention via `messageCreate` event
2. **Typing** - Send typing indicator immediately, refresh every 9 seconds
3. **Context** - Fetch last 10-15 channel messages for conversation context
4. **Memory** - Look up user's stored "goblin note" from memory file
5. **RAG** - Retrieve 3-5 relevant voice lines from dataset matching the topic
6. **Prompt** - Assemble: system prompt + user memory + voice line examples + conversation context + user message
7. **Generate** - Call Gemini Flash, request structured output:
   ```json
   {
     "reply": "You AGAIN?! Squee already told you, Squee died lots of times!",
     "memory": "Keeps asking about dying. Three times now. Squee thinks this one writes books."
   }
   ```
8. **Store** - Save updated goblin note for this user
9. **Reply** - Send response to Discord (split if >2000 chars)

## Per-User Memory System ("Goblin Notes")

Squee maintains a 1-line note per user, written in his own voice. This makes interactions feel persistent and personal.

**Example entries:**
```json
{
  "123456789": "This one keeps asking how Squee dies. Very creepy. Smells like cheese.",
  "987654321": "Nice human. Gave Squee a compliment once. Squee remembers nice things.",
  "555555555": "Talks too fast. Uses big words. Squee just nods."
}
```

- Notes are generated and updated by the LLM as part of each response (structured output).
- Memory is cumulative - Squee refines his impression of each user over time.
- ~20 users means the entire memory store fits in a single small JSON file.

## Voice Line Dataset Pipeline

### Goal: 10,000 Squee voice lines for RAG grounding

### Step 1 - Extract Authentic Source Material (~300-500 lines)

| Source | Method | Expected Yield |
|--------|--------|---------------|
| Scryfall flavor text | API query `ft:squee` | ~30 card quotes |
| Weatherlight saga novels | Manual parse for Squee dialogue | ~200-400 lines |
| MTG web fiction (Dominaria, etc.) | Extract from story archives | ~50-100 lines |
| MTG wiki summaries | Character lore compilation | Context, not dialogue |

**Key novels**: Rath and Storm, Mercadian Masques, Nemesis, Prophecy, Invasion, Planeshift, Apocalypse.

### Step 2 - Build Character Sheet

Define Squee's voice with explicit rules for the generation model:

- **Vocabulary**: ~5th grade reading level, specific words he misuses
- **Sentence structure**: Short, simple, often fragments
- **Self-reference**: Alternates "Squee" and "I" (more "Squee" when scared, more "I" when calm)
- **Topics he cares about**: Food, not dying, his friends, shiny things, being scared
- **Topics he doesn't understand**: Magic theory, politics, strategy, romance
- **Emotional range**: 70% fear/complaint, 15% wonder/excitement, 10% accidental insight, 5% genuine loyalty/bravery

### Step 3 - Generate Synthetic Voice Lines with Opus

Use Claude Opus with the character sheet + authentic examples to generate across categories:

| Category | Count | Description |
|----------|-------|-------------|
| Dialogue responses | 3,000 | Situation prompt -> Squee's reaction |
| Flavor text style | 2,000 | 1-2 sentence observations in Squee's voice |
| Conversational exchanges | 3,000 | Multi-turn dialogues with other characters |
| Narration | 2,000 | Squee describing events/places/things |
| **Total (pre-filter)** | **~10,500** | |

Output format:
```json
{
  "category": "dialogue",
  "situation": "Someone asks how many times Squee has died",
  "response": "Squee lost count after... um... what comes after eleven? Lots. Squee died lots.",
  "tags": ["death", "counting", "confusion"]
}
```

### Step 4 - Quality Filter

- Have Opus rate each line for "Squee-ness" (1-5 scale)
- Keep only 4-5 rated lines (~95% keep rate -> ~10,000 final)
- Manually spot-check ~200 lines for tone consistency
- Ensure variety: not just "dumb goblin says dumb thing" but also pathos, loyalty, accidental wisdom

### Step 5 - Store as RAG Dataset

Final `squee-voicelines.json` with tags for retrieval matching.

## Squee Character Reference

### Who Is Squee?

- **Full name**: Squee, Goblin Nabob / Squee, the Immortal / Squee, Dubious Monarch
- **Race**: Goblin from the Rundvelt warrens on Dominaria
- **Role**: Cabin boy and cook aboard the Weatherlight skyship
- **Key trait**: Literally immortal - was granted regeneration by Phyrexian experiments meant as torture, outlasted his captors

### Key Lore

- Recruited by the Weatherlight crew during the Tempest block
- Accompanied the crew to Rath to rescue Captain Sisay from Volrath
- On Mercadia, became important to the secretly-intelligent Mercadian goblins
- Captured during the Phyrexian Invasion by corrupted former crewmate Ertai
- Ertai killed Squee repeatedly as a test subject, granting him immortality to keep experimenting
- Survived for centuries after the Weatherlight saga
- By Dominaria United era, rules a goblin settlement as "Dubious Monarch"

### Personality Traits

- Cowardly but accidentally brave
- Stupid but accidentally insightful
- Perpetually hungry, scared, and confused
- Fiercely loyal to his friends (Karn, Gerrard, Hanna)
- Refers to Karn as his friend/"toy", Hanna as "the pretty lady", Gerrard as the boss
- Narrates his own terror out loud
- Complains constantly but never actually leaves

### Iconic Quotes

- "Squee don't like this."
- "Lemme see... three times three times three... lots!" *(Squee's Revenge)*
- "I got my own Karn!" *(Squee's Toy)*
- "It's nice to be needed." *(Squee, the Immortal)*
- "It's not a big step from 'annoying' to 'disposable.'" *(Squee, Goblin Nabob)*

## Build Phases

### Phase 1 - Scaffold
- Initialize TypeScript project with discord.js
- Basic bot that connects to Discord and detects @mentions
- Responds with a hardcoded Squee quote
- Deploy to Fly.io

### Phase 2 - Squee Brain
- Write the Squee system prompt (personality, rules, tone)
- Integrate Gemini Flash API
- Bot generates dynamic in-character responses
- Handle typing indicators during generation

### Phase 3 - Memory
- Implement per-user goblin notes (JSON storage)
- Structured output parsing (reply + memory update)
- Memory included in prompt context on each interaction

### Phase 4 - RAG
- Implement voice line retrieval (keyword/tag matching or simple embedding search)
- Include 3-5 relevant voice lines in each prompt as examples
- Test that responses are more grounded in Squee's authentic voice

### Phase 5 - Dataset Generation
- Extract authentic Squee material from novels/Scryfall/wiki
- Build character sheet for Opus
- Run synthetic generation pipeline
- Quality filter and store as `squee-voicelines.json`

### Phase 6 - Polish
- Per-user rate limiting (5 req/min)
- Response splitting for >2000 char messages
- Error handling (API failures, Discord disconnects)
- Graceful reconnection logic

### Phase 7 (Optional) - Fine-Tune Experiment
- Fine-tune Llama 3.2 3B on the voice line dataset via Modal/Together AI (~$5-20)
- Compare quality against Gemini Flash + RAG approach
- If better, host via Cloudflare Workers AI LoRA adapters (free) or Together AI inference (~$1-5/mo)

## Discord Bot Configuration Notes

- **Intents required**: `Guilds`, `GuildMessages`, `MessageContent` (privileged - must enable in Developer Portal)
- **Permissions**: Send Messages, Read Message History, Use External Emojis (optional for flavor)
- **Rate limits**: discord.js handles Discord API limits internally; bot-level per-user limiting is custom
- **Message limit**: 2000 characters per message - split longer responses
- **Bot vs bot**: Always check `message.author.bot` to avoid infinite loops
